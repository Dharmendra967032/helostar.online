Welcome to Python!

Python is an interpreted, high-level, general-purpose programming language. It's known for its clear syntax and readability, making it an excellent choice for beginners and professionals alike.

It's used everywhere: **web development** (Django, Flask), **data science** (Pandas, NumPy), **machine learning** (TensorFlow, PyTorch), and automation.

1. Setup and Syntax Basics

1.1 Installation and Running Code

Download the latest version from python.org. You can run Python code using an Integrated Development Environment (IDE) like **VS Code** or **PyCharm**, or directly from your terminal/command prompt.

1.2 Variables and Data Types

A variable is a container for storing data values. Python is dynamically typed, meaning you don't declare the type explicitly.

# Integer (int)
age = 30

# Floating Point (float)
price = 19.99

# String (str)
name = "Alice"

# Boolean (bool)
is_student = True

1.3 Basic Input and Output

Use the built-in functions print() for output and input() for receiving user input.

# Output
print("Hello, World!")

# Input
user_name = input("Enter your name: ")
print("Welcome,", user_name)

2. Control Flow: Decisions and Loops

2.1 Conditional Statements (if, elif, else)

Control the execution of your code based on conditions. **Indentation is mandatory** in Python to define blocks of code!

score = 85
if score >= 90:
    print("A Grade")
elif score >= 80:
    print("B Grade") # This block executes
else:
    print("C Grade or lower")

2.2 Loops (for and while)

**For loops** iterate over a sequence (like a list or a string). **While loops** execute as long as a condition is true.

For Loop Example:

fruits = ["apple", "banana", "cherry"]
for fruit in fruits:
    print(fruit)

While Loop Example:

count = 0
while count < 3:
    print("Count:", count)
    count += 1

3. Python Data Structures

Python has powerful built-in collections. Understanding these is crucial for efficient programming.

3.1 Lists (Mutable, Ordered)

Lists are ordered collections that can be modified (mutable).

my_list = [10, "hello", 3.14]
my_list.append(50) # Adds 50 to the end
print(my_list[0]) # Output: 10

3.2 Tuples (Immutable, Ordered)

Tuples are ordered collections that **cannot be changed** after creation (immutable).

my_tuple = (1, 2, "three")
# my_tuple[0] = 99 # This would cause an error!

3.3 Dictionaries (Mutable, Key-Value Pairs)

Dictionaries store data in key: value pairs and are unordered (in versions before 3.7, ordered in 3.7+).

person = {
    "name": "Bob",
    "age": 25,
    "city": "New York"
}
print(person["name"]) # Output: Bob

4. Functions: Organizing Your Code

Functions allow you to break down your program into reusable, manageable pieces.

4.1 Defining and Calling Functions

Use the def keyword to define a function.

def greet(name):
    """This function greets the person passed in as a parameter."""
    return "Hello, " + name + "!"

message = greet("Charlie")
print(message) # Output: Hello, Charlie!

Note: The text in triple quotes is a **docstring**, which explains what the function does.

5. Object-Oriented Programming (OOP)

OOP structures programs around **objects** (instances of **classes**), rather than just functions and data.

5.1 Classes and Objects

A **Class** is a blueprint, and an **Object** is an instance of that blueprint.

class Dog:
    # The constructor method
    def __init__(self, name, age):
        self.name = name
        self.age = age

    # A regular method
    def bark(self):
        return f"{self.name} says Woof!"

# Create an object (instance)
my_dog = Dog("Buddy", 5)
print(my_dog.bark()) # Output: Buddy says Woof!

5.2 Inheritance

Classes can inherit properties and methods from other classes.

class GoldenRetriever(Dog): # GoldenRetriever inherits from Dog
    def __init__(self, name, age, color):
        super().__init__(name, age) # Call parent constructor
        self.color = color

    def fetch(self):
        return f"{self.name} is fetching the ball!"

my_golden = GoldenRetriever("Goldie", 3, "golden")
print(my_golden.bark())  # Inherited method
print(my_golden.fetch()) # New method

6. Modules and Packages

Python's strength lies in its vast ecosystem of modules and packages, which are simply Python files containing functions, classes, and variables that you can use in your own code.

6.1 Importing Modules

Use the import statement to bring modules into your current script.

import math
print(math.pi) # Output: 3.14159...

from datetime import date
today = date.today()
print(today)

6.2 Creating Your Own Modules

Any .py file can be a module. For example, if you have my_module.py:

# my_module.py
def say_hello(name):
    return f"Hello, {name} from my_module!"

You can then import it in another script:

# main_script.py
import my_module
print(my_module.say_hello("World")) # Output: Hello, World from my_module!
